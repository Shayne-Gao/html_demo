Skill 化：把"做一次"沉淀为"团队资产"

把 L1–L4.5 一路积累的重复流程封装成一段可复用的 Skill——下次只要一句话触发，全流程自动跑。

1你将学会

勾选完成情况会自动同步到首页学习进度。

  • 我能用 Skill 三要素（触发 / 流程 / 产物）写出一个 Agent 能稳定复现的 Skill
  • 我能在新会话里仅用一句触发词，跑完整套场景的全流程
  • 我能识别哪些任务值得 Skill 化、哪些不值得

2背景小知识

到 L4.5，你做过的每个流程其实都"长成了一份 SOP"——L1 的 HOWTO、L2 的 README、L4.5 的 VISION_PROMPTS……

Skill 就是把它们升级一档：从"我自己看的笔记"升级为"Agent 能直接执行的工作流"。

[触发]   什么时候启动这个 Skill（关键词、文件、时间）
[流程]   分几步、每步干什么、产物路径
[产物]   完成后本地/系统里应该多出什么、形态如何

💡 一句口诀

Skill = 给 Agent 的 SOP，让它"自己看说明书做事"。
⚠️ Skill 化的判断红线：做过 ≥ 3 次、流程稳定、产物形态明确——满足这三条再 Skill 化。一次性事情不要 Skill 化，否则维护成本反而高。

⚠️如果失败怎么办
  • 偏差 A：Agent 没读 SKILL.md，直接开始动手 → 你的 SKILL.md 在写法上不够"指令式"。修正：在 SKILL.md 开头加一句"任何人（包括你自己将来的会话）调用本 Skill 时，第一件事是逐字读完本文件再开始动手"。

💡 这就是 Skill 化的本质：反复用、反复改 SKILL.md，直到任何会话都能稳定复现。

正确答案 C：Skill 化的红线是"做过 ≥3 次 + 流程稳定 + 产物明确"——一次性任务封装出来反而是负担。
Q2. Skill 三要素是？
正确答案 B：Skill 三要素是 [触发]（什么时候启动）/ [流程]（分几步、每步干什么）/ [产物]（最终落盘形态）。
Q3. Skill 化的"一档止步线"对应哪一档？
正确答案 A：本讲对应"一档 = 个人 Skill"，目标是你自己电脑里能稳定跑就够了。一档够用别上二档；二档够用别上三档。

8副线练习（可选）

本讲主线已可在三个场景切换，副线建议挑战更进阶的玩法。把你副线做过的某个流程也封装成 Skill：
