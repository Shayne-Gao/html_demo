Prompt 工程的最小够用版：4 要素模板

把"碰运气"变成"高命中率"——给 PM 一个 4 要素模板，5 分钟学会、终身受益。

1你将学会

勾选完成情况会自动同步到首页学习进度。

  • 我能用角色 / 上下文 / 约束 / 产物 4 要素模板，把任何模糊请求重写成高命中率 prompt
  • 我能识别一个 prompt 为什么会让 Agent 跑偏，并准确加上缺失的那一块
  • 我能把 L0–L2.5 用过的所有 prompt 倒过来用 4 要素拆解、复盘自己的进步

2背景小知识

到 L2.5 你已经会"使唤"Agent 了，但多半还是凭直觉。直觉的天花板就是：你不知道为什么这次结果好、那次结果差。

PM 不需要研究学术意义上的 prompt engineering。你只需要一个最小够用的 4 要素模板：

[角色]   你扮演谁
[上下文] 我在做什么、当前情境
[约束]   不能做什么 / 必须遵守什么
[产物]   我要什么形态的输出、放在哪

💡 一句口诀

角色定语气，上下文定方向，约束防跑偏，产物收住手。

大多数"Agent 不听话"，本质是这 4 块少了 1–2 块。L0–L2.5 你之所以能跑通，是因为讲义里已经偷偷在 prompt 里替你补齐了这些；本讲是把方法论交还给你。

🛠 互动模块：4 要素 Prompt Builder

左侧填 4 块，右侧实时拼接成完整 prompt。任何一块为空，右侧都会高亮提醒——亲手感受"少一块的危害"。顶栏切换场景，会自动重置成对应场景的预填示例。

4实战任务

本讲的实战不是"做一个新东西"，而是把你已经做过的事情用 4 要素重写一遍——亲手感受好 prompt 和坏 prompt 的差距。

1用一个翻车 prompt 让 Agent 翻车给你看

故意翻车是为了你记得"为什么不能这么写"。
🎯你要做什么

找一个你日常会写的"模糊请求"，故意发给 Agent。

⌨️你输入什么
帮我整理一下这周的需求池。
帮我整理一下这周的会议待办。
帮我整理一下这周的产品工作。
👀预期看到什么

Agent 会反问你或者自由发挥，结果大概率是以下之一：

  • 产出一份不知所云的"工作整理"
  • 反问你 5 个问题（而你没耐心一条条答）
  • 给一个范文模板让你"自己填"
📝 这一步要做的事：把这次回答整段复制到 pm-agent-lab/prompt-lab/before-after.md，标记为 Before。

2用 4 要素重写同一个请求

🎯你要做什么

把同样的需求，用 4 要素模板重写一次。

⌨️你输入什么
[角色]
你是我的 PM 助手，熟悉互联网产品的需求池治理场景。

[上下文]
我是一个产品经理，本周新增了三条候选需求：
1) "推荐流冷启动优化"（来源：算法同学反馈）
2) "iOS 用户登录后偶发白屏"（来源：客服转接的用户反馈）
3) "下季度 OKR 草案里提到要补一个'分享回流'入口"
对方是我的直属 leader，他时间紧，喜欢看结论，不喜欢长篇大论。

[约束]
- 输出 ≤ 200 字
- 结构：[结论一句] + [3 条需求，每条 1 行：一句话描述 + 初判优先级] + [下周 1 个最重要的下一步动作]
- 不要使用 emoji 或夸张副词（"非常"、"极大地"等）
- 我没说的不要编

⚠️如果出错怎么办
  • 偏差 A：Agent 还是写超长 → 你少写了"违反约束就重写"，纠偏：
    你刚才超过 200 字了。请把约束当硬指标：超字数立即压缩重写，不要让我提醒第二次。
  • 偏差 B：Agent 还是用了"非常"这类副词 → 让它自检：
    扫一遍你刚才的输出，列出违反"不要使用夸张副词"约束的所有词，然后改写。

3反向拆解：把 L0–L2.5 的 prompt 对号入座

🧩 这一步是 PM 学习 prompt 的"开窍时刻"

回到 L1 Step 1 里"和 Agent 一起定 schema"那段 prompt，把它拆成 4 块，写在 before-after.md 里。
📐参考拆法
要素对应原文（示意）
角色（L1 没显式写，靠会话上下文撑）
上下文"我要做一个'行业新闻周报'主线任务，每周抓一次新闻，存到本地，下游会用这些数据做 HTML 可视化。"
约束"字段尽量原子化…必含…先不要去抓真实新闻…"
产物"schema/news.schema.md…一条 mock 数据 sample.json"
📝你在拆解中会发现的事
  1. 几乎所有"好用的 prompt"都在不知不觉里凑齐了 4 要素
  2. 你之前写的 prompt 最常缺的是"约束"和"产物路径"
  3. 角色这一块 PM 经常忽略，但其实"你扮演谁"对语气和深度影响巨大

把 L0、L2、L2.5 各挑一段 prompt 重做这个拆解，每段写一两行心得。

5升级技：让 Agent 帮你"挑刺自己的 prompt"

💡 何时用：4 要素是底盘。下面这两个手法是"高级护栏"，遇到关键任务时再用。
5.1让 Agent 先 plan 再做
[在产物前面加一句]
开始动手前，先给我一份需求池整理的执行计划（你打算从哪些来源抓线索 + 用什么字段去重 + 大致顺序），让我确认后你再执行。
[在产物前面加一句]
开始动手前，先给我一份会议待办提取的执行计划（你打算读哪些纪要文件 + 怎么区分"决策/待办/风险" + 大致顺序），让我确认后你再执行。
[在产物前面加一句]
开始动手前，先给我一份执行计划（步骤 + 你打算调用什么工具 + 大致顺序），让我确认后你再执行。

用途：长任务、不可逆任务、第一次跑的新流程。

5.2让 Agent 自己"挑刺"你的 prompt
我刚刚给你的"需求池整理 prompt"你先不要执行，按下面 5 个角度挑刺：
1) 4 要素哪块缺失或不清晰？（角色/上下文/约束/产物路径）
2) 哪些表述会被你理解成至少两种意思？（比如"重要需求"具体指什么）
3) 有没有"约束太多反而互相打架"的地方？
4) 我有没有让你做一些其实你做不到 / 不该做的事？（比如要求你判断"用户是否真的会用"）
5) 如果只能保留 1 句话，最关键的是哪一句？

🧠 快速自测

Q1. 4 要素里 PM 最容易偷懒不写的是？
正确答案 D：不写产物路径 → Agent 把答案丢在对话里，下游用不了——这就是为什么对话越聊越乱。
Q2. Agent 总写超长、还动不动用"非常 / 极大地"，最该补哪个要素？
正确答案 C：字数失控、风格漂移、瞎编内容——这三类问题几乎都是"约束"那一块没写或写得太软。
Q3. PM 偷懒的极简版 prompt，至少要带哪两要素？
正确答案 C：极简版 = "做 X，约束 Y，结果落到 <路径>"——带上"约束 + 产物"，命中率就能拉到 80 分以上。

6思考题 & 下一讲引子

思考题

  • 4 要素里 PM 最容易忽略哪一块？为什么？（提示：很多 PM 不写"产物"，导致 Agent 把答案丢在对话里就完事——这就是为什么对话越聊越乱。）
  • 把 4 要素延伸成你团队所有同事都用的"团队 prompt 公约"，需要补哪一条？

下一讲引子

你已经会写好 prompt 了，但好 prompt 的"上下文"那块每次都要重写吗？显然不行。L4 我们会建一个 LLM Wiki——把"我是谁、我们产品是什么、我的偏好"一次性写好，让 Agent 永远从那里取材。

9给 PM 的小结：4 要素的"权重"

不是 4 块同等重要。按 PM 实际场景排序：

优先级要素写不好的代价
🥇产物不写 → Agent 把答案丢在对话里，下游用不了
🥈约束不写 → 字数失控、风格漂移、瞎编内容
🥉上下文不写 → 答非所问、需要反复补背景
4️⃣角色不写 → 语气与深度不对，但通常不致命

🎯 一句话带走

所以 PM 平时偷懒的极简版可以是：
"做 X，约束 Y，结果落到 <具体路径>。" ——只要带上"约束 + 产物"，命中率就能直接拉到 80 分以上。
