2背景小知识

Vibe Coding 不是一种编程语言，而是一种工作方式：

💡 一句话

你描述意图（vibe），Agent 负责实现（coding）；你不审代码，只审结果。

它对 PM 的意义是巨大的：

  • 过去：你想验证一个交互想法，要排期、找研发、几天后才看到 demo
  • 现在：你 30 分钟内就能拿到一个可点击、能筛选、能排序的 demo 挂给老板

不是要 PM 替代研发，而是让你把"想法 → 可被讨论的产物"的距离压到最短。

⚠️ 红线：Vibe Coding 出来的产物默认是一次性的——能跑、能演示、能验证想法，但不要直接拿去当生产代码。该交研发还是交研发，但交的不再是 PRD，而是已经验证过的 demo。

👀预期看到什么
  • Agent 创建 dashboard.html
  • 给你一个打开方式（双击、或起一个本地小服务器）
  • 可能踩坑：直接双击打开会被浏览器 CORS 挡住读 JSON——这是预期的，下一步处理
⚠️如果出错怎么办
  • 偏差 A：Agent 把 JSON 内嵌到 HTML 里了 → 纠偏：
    请保持 JSON 单独一个文件，HTML 用 fetch 去读，不要把数据写死在 HTML 里。这样下周新数据来时，HTML 不用动。
  • 偏差 B：Agent 引入了 Tailwind / Vue 的 CDN → 纠偏：
    不要任何外部依赖。所有样式和逻辑都内联到这个 HTML 里。我要离线双击就能用。
